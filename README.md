# 中文写作通用规则

一个用于 Codex 的中文写作与产品文案技能。它用按优先级排列的通用规则约束初稿和修改，处理主题漂移、对象突变、含糊指代、空泛复述、程式化节奏、伪造人味和过程性元叙述。

这套规则不维护高频词黑名单。模型先执行通用规则，再做一次按优先级排列的自我批评。只有用户连续否定至少两版、明确要求案例诊断，或者同一问题在自检后仍然复发时，模型才读取案例参考。案例不能覆盖事实、证据边界和作者观点。

## 安装

```bash
npx skills add tidego/chinese-prose-style-skill
```

显式调用：

```text
使用 $chinese-prose-style 修改这段中文：先应用通用规则，保留事实和作者声音，写清对象与句间关系；只有反复修改仍失败时才查案例。
```

## 通用规则检查什么

检查有明确顺序。前一项没有满足时，不用先改后一项：

1. **来源是否忠实。** 第一人称经历、情绪、场景和具体细节必须来自原稿或可靠来源；尚未发生的风险保留假设语气。
2. **结论是否越过证据。** 重要说法要区分已核实事实、有依据的推断和未决问题，权威归因必须能够追溯。
3. **全文是否围绕一个主问题。** 长文保留一个主要对象和主结论；每一节都要补足主线所需的一步，有独立价值的旁支应迁移，不应随意删除。
4. **句子是否完成对象交接。** 行动者、时间、条件、业务阶段、分支或证据状态变化时，写明真实关系。关系无法直接读出时才使用准确的连接词，不因主语变化机械添加“然后”或“同时”。
5. **每句话是否增加信息。** 关键证据写清谁在什么条件下做了什么；段尾若只把前文改称为某种“路径”“机制”或“能力”，应直接删除。
6. **正文是否保持成品视角。** 删除与主题无关的调研障碍、修改记录、章节预告和措辞选择。产品文案只描述用户可见的对象、状态、动作或结果。
7. **中文语序是否自然。** 翻译材料保留原文事实和术语，不照搬英文语序；行动者明确时优先使用主动表达，但不能补出来源没有提供的主体或因果。
8. **术语和语言是否一致。** 同一对象固定一种叫法，中文正文不夹入无必要的英文连接语或普通说明语；产品名、论文名、代码标识和没有稳定中译的技术术语保留原文。
9. **形式是否服务信息。** 排比、对称句、列表和比喻只有在结构本身承载信息时才保留；相邻章节不应重复同一模板，比喻也不能扩张成整段映射。
10. **语气是否来自作者。** 不用强硬判断代替观点，不用口语、反问、短句或虚构情绪表演“像真人”。

“它”“这条路径”“该机制”“这一点”和“其他逻辑”不是禁词。只有前文存在唯一、紧邻的所指，而且这些表达没有隐藏行动者、条件或证据时，才可以保留。关键位置重复准确名词，通常比让读者回头猜测更清楚。

本技能不会机械禁用破折号、冒号或某个高频词。单个形式不是证据，反复出现且没有信息作用时才需要修改。

完整的设计背景和研究资料见[中文“AI 味”是怎样被训练出来的？](https://guzhaolin.art/blog/chinese-ai-prose-style)。

## 能力边界

本技能只在推理时提供约束，不会改变模型参数。它不能证明一段文字是否由模型生成，也不能替代作者的事实、经历和判断。目前没有正式基准证明它能稳定提升所有中文文体；仓库因此不声明确定的效果增益。
