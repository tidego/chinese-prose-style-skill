# 中文写作通用规则

一个用于 Codex 的中文写作与产品文案技能。它用按优先级排列的通用规则约束初稿和修改，处理主题漂移、对象突变、含糊指代、空泛复述、程式化节奏、伪造人味和过程性元叙述。

这套规则不维护高频词黑名单。模型先执行通用规则，再做一次按优先级排列的自我批评。只有用户连续否定至少两版、明确要求案例诊断，或者同一问题在自检后仍然复发时，模型才读取案例参考。案例不能覆盖事实、证据边界和作者观点。

## 安装

```bash
npx skills add tidego/chinese-prose-style-skill
```

显式调用：

```text
使用 $chinese-prose-style 修改这段中文：先应用通用规则，保留事实和作者声音，写清对象与句间关系；只有反复修改仍失败时才查案例。
```

## 通用规则检查什么

- 第一人称经历、情绪和场景是否有来源；
- 假设风险是否被误写成已经发生的事件；
- 长文是否只有一个主问题、主要对象和主结论；
- 每一节是否为主结论所必需，有独立价值的旁支是否被妥善迁移；
- 具体案例和权威归因是否能追溯到来源；
- 重要结论是否区分事实、推断和未决问题；
- 行动者、业务阶段或证据状态变化时，句子是否说明了真实关系；
- 关键证据是否依赖含糊代词或抽象指代；
- 段尾是否只用抽象名词复述前文；
- 正文是否残留调研障碍、修改记录、章节预告或措辞选择；
- 产品文案是否描述用户对象和状态，而不是开发计划或智能体汇报；
- 排比和对称句是否提供了真实信息；
- 抽象名词是否省略了行动者和动作；
- 比喻是否准确对应某个机制，还是只增加结论感；
- 一个比喻是否被继续拆成整段映射，相邻章节是否重复同一模板；
- 情绪强度是否来自作者原稿；
- 口语、反问和短句是否在表演“像真人”。

本技能不会机械禁用破折号、冒号或某个高频词。单个形式不是证据，反复出现且没有信息作用时才需要修改。

完整的设计背景和研究资料见[中文“AI 味”是怎样被训练出来的？](https://guzhaolin.art/blog/chinese-ai-prose-style)。

## 能力边界

本技能只在推理时提供约束，不会改变模型参数。它不能证明一段文字是否由模型生成，也不能替代作者的事实、经历和判断。目前没有正式基准证明它能稳定提升所有中文文体；仓库因此不声明确定的效果增益。
